The Bazaar needs to have a fully wired up Add Marketplace & Add Plugin system which you input a github repo URL and it pulls the marketplace.json / plugin.json and all files + metadata and allows you to view / browse all the Marketplace plugins + browse all of the plugin's components, like agents, commands, skills, hooks, etc.
Lets get rid of the dual tab system, the unified sidebar should be:
- Hub - A dashboard for all of your published artifacts, marketplaces, plugins, agents, commands, skills, hooks, scripts, outputs, MCPs, LSPs, settings.json files, README.md files, notes, and TODOs. Displays stats, recent activity, notifications, and general information about your account and projects.
- Feed - View all of the latest news, updates, and announcements from starred sources. Trendings, New Releases, and Announcements.
- Artifacts - View all of your artifacts, organized by tags and categories. Agents, Commands, Skills, Hooks, Scripts, Output-styles, .mcp.json, .lsp.json, settings.json files, README.md, Plans, Sessions, Reports, Research, and Specs files. Create new artifacts and edit existing ones.
- Projects - A repo-scoped Stash for all of your project-specific artifacts, notes, and TODOs. For Projects - you can obviously view all repo-scoped Artifacts, but mainly what we're interested in creating with Projects is a Hub FOR your agents to use - we'd have a Project Feed, where Agents will check in to post reports, status check ins, task tracking, new plans created, recent session files; along with the ability to link any artifact, note, or TODO to a project. This way you can easily see all of the relevant information for a project in one place. You can also create project-specific bundles and share them with your team. So Projects is really meant to be a hub for all of your non-plugin related Artifacts - the Artifacts here are what we generate: Ideas, Notes, TODOs, Beads, Plans, Sessions, Reports, Research, and Specs files. Obviously there's several Artifacts that can be both project-specific and plugin-specific, like Agents, Commands, Skills, Hooks, Scripts, Output-styles, MCPs, LSPs, settings.json files, and README.md files. So those can be created in either the Artifacts or Projects section and linked to the other as needed. Projects is meant to be a shared space for Agents to post updates, reports, and information about a specific project, allowing you to easily track progress, review plans, and to quickly and easily access relevant information for all of your projects in one place without having to hop from machine to machine via terminal or slack channels.
- Bazaar - Browse and install Marketplaces, Plugins, Agents, Commands, Skills, Hooks and Scripts from GitHub repos. View/Download all of the metadata, files, and components of each Marketplace and Plugin.
- Drafts - View all of your drafts, organized by tags and categories. Create new drafts and edit existing ones. Drafts can be converted into published artifacts, marketplaces, plugins, agents, commands, skills, hooks, scripts, output-styles, MCPs, LSPs, settings.json files, README.md files, notes, and TODOs.
- Ideas - Canvas for brainstorming and organizing your ideas. Link any number of your artifacts together. Create mind maps, flow charts, and diagrams to visualize your ideas and how they connect to each other.
- Creator - Create a Bundle, Marketplace, Plugin, Agent, Commands, Skill, Hook, Script, Output-style, MCP, LSP, settings.json, README.md
- Notes - View all of your notes, organized by tags and categories. Create new notes, edit existing ones, and link them together for easy reference.
- TODOs - Keep track of your tasks and to-dos. Create new tasks, set due dates, and mark them as complete when finished.
- Beads - View all of your beads https://gastownhall.github.io/beads/
- Bundles - bundle any marketplace, plugin, artifacts into a curated collection of custom tools to share with your team. Able to install a bundle to your agent.
